Pattern for multi-instance pages

When the wordmark appears more than once on a page (e.g. nav + footer), define it ONCE via <symbol> and reuse it. Every outer <svg> that does this MUST carry viewBox="0 0 110.72 23.37" - without it, browsers default to a 300×150 box and centre the wordmark inside, making the logo look like it has a wide invisible gutter around it. The <symbol>'s own viewBox doesn't help here - it only governs the symbol's internal coordinates, not the outer SVG element's size.

<!-- Place once at top of body --> <svg width="0" height="0" style="position:absolute" aria-hidden="true"> <defs> <symbol id="wp-wordmark" viewBox="0 0 110.72 23.37"> <path d="M28.4,7.47c-1.32,0,... [full path data] ..."/> </symbol> </defs> </svg> <!-- Reuse anywhere on the page - viewBox is REQUIRED on every outer <svg> --> <a href="#top" class="nav-logo"><svg viewBox="0 0 110.72 23.37" style="fill:#070909"><use href="#wp-wordmark"></use></svg></a> <div class="footer-logo"><svg viewBox="0 0 110.72 23.37" style="fill:#fff"><use href="#wp-wordmark"></use></svg></div>

Voice + writing

Rules
RuleDetail
Sentence case headlinesOnly capitalise the first word and proper nouns. "Branded search and shopping drove a strong month" - not "Branded Search And Shopping Drove A Strong Month".
Australian Englishorganise, optimise, colour, centre, behaviour, analyse, programme. NEVER organize / optimize / color / center.
Numerals for numbers"20 years" not "twenty years". "4.5x ROAS" not "four-and-a-half times". "$278K" not "two hundred and seventy-eight thousand dollars".
No em dashesUse " - " (space-hyphen-space). Never the em-dash character (Unicode U+2014). This is the most common authoring mistake.
Excluded brandsSee SKILL.md for the full exclusion list. Excluded brands must NEVER appear in any deliverable, even as examples.
Footer taglineVerbatim: "We believe there's always a smarter, faster + better way of doing things". No edits. Smart quotes optional.
Period labelsUppercase with no comma: "FEB 2026", "JAN-MAR 2026". Spelled-out forms only inside body text.

Do / don't

Visual reference

Section header

Do · kit canonical

Key metrics

February 2026
Title and red eyebrow inline on the same baseline.
Don't · stacked legacy
FEBRUARY 2026

Key metrics

Eyebrow sitting above title - reads heavier and dates the report visually.

Background colour

Do · warm off-white
#F7F6F5Has yellow/red undertone. Reads as paper, not screen.
Don't · cool blue-grey
#F4F6F8Reads as software UI. NEVER use blue-grey backgrounds.

Headline case

Do · sentence case
Branded search drove a strong monthOnly first word + proper nouns capitalised.
Don't · title case
Branded Search Drove A Strong MonthEvery word capitalised. Looks like a press release headline.

Numeric alignment in tables

Do · .num class + tabular-nums
Performance Max$148,2105.12x
Shopping$66,2894.70x
Branded search$52,9304.27x
Numerals line up vertically. Decimal points align. Easy to scan.
Don't · default font + left-align
Performance Max$148,2105.12x
Shopping$66,2894.70x
Branded search$52,9304.27x
Variable-width numerals don't align. Hard to compare at a glance.

Em dash vs hyphen

Do · space-hyphen-space
Revenue grew on lower spend - the system is working more efficiently.Plain hyphen with single spaces on either side.
Don't · em dash
Revenue grew on lower spend [EM-DASH HERE] the system is working more efficiently.Smart auto-replace from word processors. Looks formal/literary.
